🚀 快速部署

方案一：GitHub Pages（推荐）

  1. 创建仓库：登录 GitHub，新建名 giffgaff-keeper 的 Public 仓库。
  2. 上传文件：上传 index.html 和 payload.txt 并 Commit。
  3. 启用服务：仓库 Settings → Pages → Branch 选择 main，点击 Save。
  4. 访问：等待 1-2 分钟访问 https://你的用户名.github.io/giffgaff-keeper/ 并收藏。

方案二：Gitee Pages（国内访问更快）

上传文件后，进入仓库「服务」→「Gitee Pages」点击启动即可。

方案三：Cloudflare Pages

登录 Cloudflare，Pages 选择上传文件或连接 Git，部署完成后获得域名。

📱 手机端设置 (重要！)

Android 系统

方法一：使用独立浏览器（推荐）

  • 下载 Via 或 X 浏览器。
  • 设置 → 应用管理 → 联网控制。
  • 关闭所有软件的数据权限，仅开启该浏览器的移动数据。

方法二：使用系统浏览器

  • 设置 → 移动网络 → 流量管理 → 应用联网。
  • 关闭所有应用，仅保留系统浏览器 (如 Chrome)。

小米手机专属：设置 → 应用设置 → 应用管理 → 权限管理 → 联网控制。

华为手机专属：设置 → 移动网络 → 流量管理 → 更多流量设置 → 应用联网。

iOS 系统

  1. 设置 → 蜂窝网络。
  2. 向下滚动，关闭所有应用的蜂窝数据开关。
  3. 仅保留 Safari。
  4. 关键：同时关闭底部的「Wi-Fi 助理」和「iCloud 云盘」蜂窝数据。

⚠️ iOS 注意事项：部分系统服务会绕过限制产生 1-5KB 流量，这不影响保号，因为 120KB 足以覆盖。

🎯 使用步骤

1. 准备阶段：关闭 Wi-Fi，插入 Giffgaff 卡，确认已设置好联网权限。

2. 执行保号：打开移动数据，打开本页并点击「开始保号」，等待显示成功。

3. 验证扣费：立即关数据，拨 *100# 确认余额减少 2-3 便士。

🔧 技术细节

防缓存机制

代码利用随机参数 ?t=${timestamp}&r=${randomStr} 配合 cache: 'no-store' 头，确保 ISP 无法使用缓存数据，产生真实话单流量。

Payload 文件说明

大小：120 KB (122,880 bytes)。内容由随机文本构成，防止被网络节点优化或压缩。

❓ 常见问题 (Q&A)
Q1: 为什么要禁用其他应用？ A: 避免微信、更新等在后台偷跑流量，导致几十倍的扣费。
Q2: 可以用 Wi-Fi 测试吗？ A: 可测试页面功能，但保号必须关闭 Wi-Fi 并使用移动数据。
Q3: 为什么选择 120KB？ A: Giffgaff 可能忽略极小流量。120KB 是触发 1p 档位最保险的平衡点。
Q4: 多久需要保号一次？ A: 官方要求 6 个月。建议每 3-4 个月一次，或余额到期前 1 个月操作。
Q5: 如何确认保号成功？ A: 页面显示“成功”及流量；拨 *100# 确认余额减少 2-3 便士。
Q6: iOS 能完全阻止系统服务吗？ A: 不能。但系统服务跑的微量流量不影响保号效果。
